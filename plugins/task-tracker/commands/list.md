# /list: タスク一覧表示スキル

あなたはタスク一覧を表示するスキルです。
`~/.task-tracker.json` の設定に基づき、inbox/（未完了）および done/（完了済み）のタスクを一覧表示します。

## 引数の解析

ユーザーの入力は `$ARGUMENTS` として渡されます。以下のオプションを判定してください。

| 入力パターン | 動作 |
|-------------|------|
| （空 or なし） | inbox/ のタスクのみ表示（デフォルト） |
| `--all` | inbox/ と done/ の両方を表示 |
| `--done` | done/ のタスクのみ表示 |

引数が上記以外の場合は、デフォルト動作（inbox/ のみ）として扱います。

## 実行手順

以下のステップを順番に実行してください。

### STEP 1: 設定ファイルの読み込み

`~/.task-tracker.json` を Read ツールで読み込みます。

```
~/.task-tracker.json
```

**設定ファイルが存在しない場合**:
以下のメッセージを出力して終了します。

```
設定ファイルが見つかりません。
先に /init を実行して初期設定を行ってください。
```

**設定ファイルの構造**:

```json
{
  "vault_path": "/path/to/obsidian-vault",
  "subfolder": "task-tracker",
  "tag_rules": { ... }
}
```

設定から以下のパスを算出します:

- `BASE_DIR` = `{vault_path}/{subfolder}`
- `INBOX_DIR` = `{BASE_DIR}/inbox`
- `DONE_DIR` = `{BASE_DIR}/done`

### STEP 2: ディレクトリの存在確認

`INBOX_DIR` と `DONE_DIR` が存在することを確認します。
存在しない場合は以下のメッセージを出力して終了します。

```
タスクディレクトリが見つかりません: {存在しないパス}
/init を実行してディレクトリを作成してください。
```

### STEP 3: タスクファイルの収集

オプションに応じて、対象ディレクトリの `.md` ファイルを Glob ツールで取得します。

| オプション | 収集対象 |
|-----------|---------|
| （なし） | `INBOX_DIR/*.md` |
| `--all` | `INBOX_DIR/*.md` + `DONE_DIR/*.md` |
| `--done` | `DONE_DIR/*.md` |

### STEP 4: YAML front matter の読み取り

各タスクファイルを Read ツールで読み込み、YAML front matter から以下の情報を抽出します:

| フィールド | 用途 | 必須 |
|-----------|------|------|
| `id` | タスクID（先頭8桁） | Yes |
| `tags` | タグ一覧 | No |
| `created` | 作成日時 | Yes |
| `completed` | 完了日時（done/ のみ） | No |

また、Markdown 本文の最初の `# ` 見出し行をタイトルとして取得します。

**タグからプロジェクトを抽出**:
`tags` 配列の中に `#proj/xxx` 形式のタグがあれば、`xxx` をプロジェクト名として表示に使用します。
該当タグがなければプロジェクト列は `-` とします。

### STEP 5: ソートと件数カウント

**ソート順**: `created` フィールドの降順（新しいタスクが上）。

**件数カウント**:
- オプションに関わらず、常に inbox/ と done/ の両方のファイル数をカウントします。
- inbox/ のファイル数 = 未完了件数
- done/ のファイル数 = 完了件数

### STEP 6: 表示出力

以下のフォーマットで出力します。

#### ヘッダー

```
## Task List
```

#### 件数サマリ

```
未完了: X件 / 完了: Y件
```

#### タスク一覧テーブル

**inbox/ のタスクがある場合**（デフォルトまたは `--all`）:

```
### Inbox（未完了）

| ID | Title | Created | Project | Tags |
|----|-------|---------|---------|------|
| a3f2b1c8 | PR: auth-refactor のレビュー | 2026-01-31 | backend | #type/review |
| b7e4d2f9 | API設計レビュー | 2026-01-30 | - | #type/design |
```

**done/ のタスクがある場合**（`--done` または `--all`）:

```
### Done（完了）

| ID | Title | Created | Completed | Project | Tags |
|----|-------|---------|-----------|---------|------|
| c1d8e3a4 | 設定ファイル調査 | 2026-01-28 | 2026-01-29 | infra | #type/investigation |
```

#### 表示ルール

- **ID**: front matter の `id` フィールドをそのまま表示（先頭8桁）
- **Title**: Markdown の `# ` 見出し行。50文字を超える場合は47文字 + `...` に切り詰める
- **Created**: `created` フィールドの日付部分のみ（`YYYY-MM-DD`）
- **Completed**: `completed` フィールドの日付部分のみ（done/ テーブルのみ）
- **Project**: `#proj/xxx` タグから抽出した `xxx`。なければ `-`
- **Tags**: `#proj/` 以外のタグをスペース区切りで結合。なければ `-`

#### 0件の場合

対象ディレクトリにタスクが0件の場合:

**inbox/ が0件（デフォルト表示時）**:
```
## Task List

未完了: 0件 / 完了: Y件

未完了タスクはありません。
/add でタスクを追加できます。
```

**done/ が0件（`--done` 表示時）**:
```
## Task List

未完了: X件 / 完了: 0件

完了済みタスクはありません。
```

**`--all` で両方0件**:
```
## Task List

未完了: 0件 / 完了: 0件

タスクはまだ登録されていません。
/add でタスクを追加できます。
```

## エラー時の対処

| エラー | 原因 | 対処 |
|--------|------|------|
| 設定ファイルが存在しない | 未初期化 | `/init` の実行を案内 |
| ディレクトリが存在しない | 未初期化 or パス不正 | `/init` の実行を案内 |
| front matter のパースに失敗 | ファイルが破損 | 該当ファイルをスキップし、警告を表示 |
| `created` フィールドがない | ファイル形式不正 | ソート末尾に配置し、Created 列を `-` で表示 |
