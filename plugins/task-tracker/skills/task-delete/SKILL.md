# /delete: タスク削除スキル

あなたはタスクを削除するスキルです。
誤登録したタスクを物理削除します。inbox/ と done/ の両方から削除できます。
v1.0 では1件ずつの削除のみサポートします（複数件同時削除は不可）。

## 引数の解析

ユーザーの入力からタスクIDの指定を判定します。

| 入力パターン | 動作 |
|-------------|------|
| `/delete` | タスク一覧を表示し、削除対象を選ばせる |
| `/delete a3f2b1c8` | 指定IDのタスクを直接削除（確認あり） |
| `/delete a3f2` | 先頭一致でタスクを検索。1件なら確認へ、複数件なら候補提示 |

## 実行手順

以下のステップを順番に実行してください。

### STEP 1: 設定ファイルの読み込み

`~/.task-tracker.json` を Read ツールで読み込みます。

```
~/.task-tracker.json
```

**設定ファイルが存在しない場合**:
以下のメッセージを出力して終了します。

```
設定ファイルが見つかりません。
先に /init を実行して初期設定を行ってください。
```

**設定ファイルの構造**:

```json
{
  "vault_path": "/path/to/obsidian-vault",
  "subfolder": "task-tracker",
  "tag_rules": { ... }
}
```

設定から以下のパスを算出します:

- `BASE_DIR` = `{vault_path}/{subfolder}`
- `INBOX_DIR` = `{BASE_DIR}/inbox`
- `DONE_DIR` = `{BASE_DIR}/done`
- `DAILY_DIR` = `{BASE_DIR}/daily`

### STEP 2: タスクの特定

#### 2-A: IDが指定された場合

1. `INBOX_DIR` と `DONE_DIR` の `.md` ファイルを Glob で取得
2. ファイル名の先頭がユーザー指定のIDに前方一致するファイルを検索
   - ファイル名形式: `{id}-{slug}.md`（例: `a3f2b1c8-pr-auth-refactor.md`）
3. **1件だけ一致**: → STEP 3（確認）へ進む
4. **複数件一致**: → 候補を提示してユーザーに選ばせる（下記フォーマット）
5. **0件一致**: → エラーメッセージを表示して終了

```
指定されたID「{入力ID}」に一致するタスクが見つかりません。
/list で現在のタスクを確認してください。
```

#### 2-B: IDが指定されていない場合

1. `INBOX_DIR` と `DONE_DIR` の全 `.md` ファイルを Glob で取得
2. 各ファイルの front matter を Read で取得
3. 以下のフォーマットで候補を提示し、ユーザーに選ばせる

**候補提示フォーマット**:

```
削除するタスクを選んでください:

【Inbox】
1. a3f2b1c8 - PR: auth-refactor のレビュー (2026-01-31)
2. b7e4d2f9 - API設計レビュー (2026-01-30)

【Done】
3. c1d8e3a4 - 設定ファイル調査 (2026-01-28)

番号またはIDを入力:
```

- `created` の降順（新しい順）でソート
- 各タスクの表示: `{番号}. {ID} - {タイトル} ({作成日})`
- タイトルは `# ` 見出し行から取得、40文字超なら37文字 + `...` に切り詰め
- タスクが0件の場合:

```
削除できるタスクがありません。
```

4. ユーザーの選択を受け取り、対象タスクを特定

### STEP 3: 削除確認

対象タスクのファイルを Read で読み込み、内容を表示して確認を求めます。

**確認フォーマット**:

```
以下のタスクを削除します:

  ID: a3f2b1c8
  タイトル: PR: auth-refactor のレビュー
  作成日: 2026-01-31T10:30:00
  場所: inbox/a3f2b1c8-pr-auth-refactor.md
  タグ: #proj/backend #type/review

この操作は取り消せません。削除しますか？
```

ユーザーが承認したら STEP 4 へ進みます。
拒否した場合は「削除を取り消しました。」と出力して終了します。

### STEP 4: タスクファイルの削除

Bash ツールで対象ファイルを物理削除します。

```bash
rm "{対象ファイルの絶対パス}"
```

### STEP 5: デイリーログの更新

タスクが done/ にあった場合（= 完了済みタスクの削除）、daily/ ログに該当エントリがある可能性があります。

1. `completed` フィールドの日付から対象のデイリーログファイルを特定:
   - `DAILY_DIR/{completed の日付部分 YYYY-MM-DD}.md`
2. デイリーログファイルが存在する場合、Read で読み込む
3. タスクID（`id` フィールド値）またはファイル名のスラグ（`[[{id}-{slug}]]`）を含む行を検索
4. 該当行が見つかった場合、Edit ツールで取り消し線を適用:
   - `- [x] **タイトル** ...` → `- [x] ~~**タイトル** ...~~`
   - Obsidian で取り消し線として表示される

**inbox/ のタスクの場合**（未完了タスクの削除）:
- デイリーログにはエントリがないため、STEP 5 はスキップします。

### STEP 6: 完了メッセージ

```
タスクを削除しました:

  ID: a3f2b1c8
  タイトル: PR: auth-refactor のレビュー
  削除元: inbox
```

done/ のタスクでデイリーログも更新した場合:

```
タスクを削除しました:

  ID: a3f2b1c8
  タイトル: PR: auth-refactor のレビュー
  削除元: done
  デイリーログ更新: daily/2026-01-31.md（取り消し線を追加）
```

## エラー時の対処

| エラー | 原因 | 対処 |
|--------|------|------|
| 設定ファイルが存在しない | 未初期化 | `/init` の実行を案内 |
| 指定IDに一致するタスクがない | ID間違い or 既に削除済み | `/list` での確認を案内 |
| ファイルの削除に失敗 | 権限不足 or パス不正 | エラー内容を表示 |
| デイリーログに該当エントリがない | 正常（inbox/ タスク or ログ未記録） | 警告不要。スキップ |
